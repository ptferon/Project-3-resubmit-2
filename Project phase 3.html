Lesson 1: Object Oriented Programing

  • Object oriented programing is the idea of breaking programing into smaller "ojects" and writing code to in the form a procedures/functions which are call methods.
  • By breaking the problem or program into smaller pieces it makes it easier for programmers to desing a program or solve a problem.
  • Each objects can interact with other object to build the program.
  • Abstract thinking: HTML uses div tags to break a website into "boxes". Each box can be designed and opporate differently. When you put all of the "boxes" together you have the structure or framework of your webpage.
  • Abstract thinking: CSS makes use of "classes" to add style to an HTML structure. Much like "objects" in OOP "classes" is CSS can be reused. In both cases programmers can avoid repetition, mistakes, and edit in a more efficient manner.
  • OOP is important to programers because OOP is focused on the data (Object) that gets the intended output. Programmers can identify objects needed to solve a problem and design the object to solve that particular problem. These objects work independtly to solve their own problem and at the same time interact with each other and work together to solve the "bigger" task at hand
  • Abstract thinking: Building a new home requires objects like Structure/framing, electrical, plumming, HVAC, and landscapping. A designer can build each independantly with its own set of responsibilties. Yet they all need to work toghether to achieve the ultimate goal of building the house.
  • Breaking the constuction/program into objects allows the designer to focusing on one "object" say the HVAC; the programmer/designer may be able to reuse a design or "object", perhaps from a different job, thus saving time and resources

Lesson 4: Introduction to Classes

  • A Class is like blue print. It provides the structure or rules by which you can use it. The class contains the data and methods which describe what the class will do.
  • For example: Class Turtle will have methods that allow the turtle to move in different directions, choose a size, and choose a color.
  • You can create multiple instances of a class. For example we created two instances of the turtle, brad and angie. Both had access to the methods with in turtle that allowed us to do different things with each turtle.
  • With in a class is the def init function. When this function gets call the init function will create space in memory for the different instances of the class that you generate.
  • An object is an instance of a class. So when we created Brad we actually created an "object" of turtle.Turtle
  • So brad = turtle.Turtle → Brad became an Object of turtle. Class Turtle() calls the def__init__() function and space is created in memory for Brad.
  • def__init__ is called the contsructor. It uses self or the instances being created. Self is the object being created.
  • Instance Variables are all of the variables associated with the instance. In our movie website creation self.title = movie_title. This was an instance variable for what ever movie we would call. For example toy_story.movie.title; self→ toy_story and the instance varible called was self.title which we assigned to movie_title. When we input the "Toy Story" for the argument movie_title the result of print toy_story.movie.title→ "Toy Story"
  • Instance Methods Are all of the functions inside the class that have self as there first argument. For exampe def show_trailer(self) was an instance method of the Class Movie we created. This instance method had instance variables webbrowser.open(self.trailer_youtube_url)
  • Class variables are variables that all instances of a class can share.
  • class variables are outside of the __init__ function which allows them to be shared among all instances of a class.
  • Since they are a constant variable we use all caps. In our class Movie we used the class variable VALID-RATINGS["G","PG", "PG-13", "R"] This allowed us to access the variable with having to create an instance variable valid_ratings for each instance of the class.

Lesson 5: Inheritance

  • Inheritance is when you inherit certian instance variables (attributes) from a parent class.
  • when you want to inherit from a parent class you place the name of the parent class in your argument: For exampe class Child(Parent)
  • When you do this you then inherit all of the instance variables
  • Additional when you create the __init__ function for the child class you put Parent.__init__(self, arg 1, arg 2...)
  • Method Overwritting is the ability of a sub class to over write a method that was inherited from the parent class
  • Inheritance is important because it helps us reuse code and become more proficient programmers!
